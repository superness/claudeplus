# Template Name Field Fix

## Problem Identified

The standalone pipeline HTML page was showing templates as "undefined" when loading them.

## Root Cause

**Mismatch between template JSON field and JavaScript expectations:**

- **JavaScript code** (`standalone-pipeline.js` line 1139, 1144): Expects `template.name`
- **Refactored templates**: Used `pipeline_name` field instead of `name`
- **Original templates** (bug-fix-v1.json, etc.): Used `name` field ✅

## Example of the Issue

**Refactored template (WRONG):**
```json
{
  "pipeline_name": "full-mining-cycle",  ← JavaScript can't find this
  "version": "2.0.0",
  ...
}
```

**Expected by JavaScript (CORRECT):**
```json
{
  "name": "full-mining-cycle",  ← JavaScript looks for this
  "version": "2.0.0",
  ...
}
```

## Solution Applied

Changed all refactored templates from `pipeline_name` to `name` field:

```bash
sed -i 's/"pipeline_name":/"name":/' [all 8 mining templates]
```
