## III. EXPLOITS FOUND

### A. CRITICAL: Daily/Weekly Cap Bypass

```
EXPLOIT: INCOME CAP INEFFECTIVE

Mechanism:
├── Design: "Hourly drop cap: 500g (soft cap)"
├── "Daily quest gold cap: 2,000g"
├── "PvP daily cap: 1,000g"
├── Total implied daily cap: ~4,000g

Actual Income Sources:
├── Quest rewards: Not all quests count toward cap
├── Enemy drops: 500g/hr × 8 hours = 4,000g
├── Vendoring items: Uncapped
├── World bosses: Uncapped
├── Group content bonuses: Uncapped
├── Weekly bonuses: Adds 3,000g
├── Trading profits: Uncapped (zero-sum anyway)

Result: Dedicated players earn 2-3x stated caps

SEVERITY: MODERATE
- Not a true exploit (working as coded)
- But income projections are underestimating reality
- Causes wealth accumulation faster than designed
```

**RECOMMENDATION:**
```
INCOME CAP FIXES

Option A: Comprehensive Daily Cap
├── All gold sources share one pool: 6,000g/day hard cap
├── Includes: quests, drops, vendoring, boss rewards
├── Excludes: trading (zero-sum), weekly bonuses

Option B: Diminishing Returns (Preferred)
├── First 4,000g/day: 100% rate
├── 4,001-6,000g: 50% rate
├── 6,001-8,000g: 25% rate
├── 8,001+: 10% rate
├── Preserves hardcore feeling without runaway accumulation
```

### B. MODERATE: Teef Conversion Arbitrage

```
EXPLOIT: ORK CURRENCY LOOP

Mechanism:
├── Ork earns 1000 Teef
├── Converts to Thrones: 1000 × 1.2 = 1,200g
├── Trades with Imperial player for items
├── Imperial sells items for Thrones
├── Imperial converts to... wait, can't convert back

ACTUALLY: Not an exploit - one-way conversion
├── Teef → Thrones works
├── Thrones → Teef = 0.9 rate (loss)
├── No profitable loop exists

VERDICT: FALSE ALARM
- Design accounts for this correctly
- 10% spread prevents arbitrage
```

### C. MODERATE: Gambling Den Exploitation

```
EXPLOIT: GAMBLING PROBABILITY MANIPULATION

Mechanism (Potential):
├── Squig racing outcomes
├── If outcomes determinable/predictable...
├── Players could identify patterns
├── Guaranteed profit over time

Analysis:
├── Design states "House edge: 5%"
├── If RNG is proper, long-term players lose 5%
├── This is GOOD (gold sink)
├── If RNG is flawed, exploit possible

REQUIREMENT: Verify RNG Implementation
├── Must use cryptographically secure RNG
├── No pattern prediction possible
├── Audit trails for suspiciously lucky players

VERDICT: POTENTIAL EXPLOIT (implementation-dependent)
```

### D. LOW: Material Vendor Arbitrage

```
EXPLOIT ATTEMPT: Vendor Material Flip

Mechanism:
├── Buy T2 materials: 25g each (20/day limit)
├── Craft T3 item requiring 4× T2 mats = 100g materials
├── T3 item vendor value: 200g sell price?

Analysis:
├── T3 equipment vendor sell price: 200g (to player)
├── T3 equipment vendor buy price: 200g? No - spread exists
├── Player sells TO vendor: 200g (per design doc)

CHECK MATH:
├── Materials bought: 100g
├── Crafting fee: 10% of output = 20g (if 200g output)
├── Total cost: 120g
├── Vendor sells for: 200g... wait, that's wrong

DESIGN DOC ISSUE FOUND:
├── "T3 Equipment | Player Sells: 200g | Player Buys: 1,200g"
├── Player sells TO vendor for 200g
├── Craft cost: 100g materials + 20g fee = 120g
├── Profit: 80g per craft

VERDICT: MILD ARBITRAGE EXISTS
├── Limited by material purchase caps (20/day)
├── Maximum daily profit: ~400g
├── Not game-breaking, but unintended

RECOMMENDATION: 
├── Increase crafting fee to 15-20% for T3
├── Or reduce T3 vendor buy price to 100g
```

### E. CRITICAL: Reserved Listing Remnant Check

```
VERIFICATION: Reserved Auction Removal

Design States:
├── "REMOVED: Reserved Listings (was exploitable for RMT)"
├── "All listings visible to all players"
├── "No private/reserved auctions"

Checking for remnants:
├── Mail system: Still allows item transfer ✓ GATED
├── Direct trade: Still allows item transfer ✓ GATED  
├── AH: No reserved option ✓ GOOD

Potential Bypass:
├── Player A lists garbage item for 50,000g
├── Player B (RMT buyer) purchases garbage item
├── Result: Gold transferred, but...
├── Protection: "Listings >300% regional average flagged"
├── Garbage items have no regional average? 

LOOPHOLE FOUND:
├── Novel items with no price history
├── No "regional average" to compare against
├── Could be used for RMT transfers

SEVERITY: MODERATE
├── Requires 12.5% fee sacrifice (costly for RMT)
├── But still possible

RECOMMENDATION:
├── All items must have "estimated value" 
├── Based on material components or tier baseline
├── Flag listings >300% of estimated value
```

## IV. BALANCE ISSUES

### A. Reward Rate Issues

```
ISSUE 1: Early Game Rewards Too Low

Problem:
├── New player earns ~150g/hour
├── T2 weapon costs 250g (vendor)
├── T2 weapon costs ~150g (player market)
├── Time to first upgrade: 1-2 hours

This is actually FINE - provides immediate goal

ISSUE 1 STATUS: FALSE ALARM
```

```
ISSUE 2: Mid-Game Valley

Problem:
├── Level 20-30 players
├── T3 gear requires significant gold
├── T3 set: ~8,000g (player prices)
├── Weekly income: ~6,000g
├── Time to T3 set: 8-10 weeks?

Design claims:
├── "T3 gear set: Casual 2 weeks, Moderate 1 week"
├── This requires ~8,000g in 1-2 weeks
├── Casual income: 6,000g/week
├── Moderate income: 13,000g/week

MATH CHECK:
├── Moderate 13,000g × 1 week = 13,000g ✓
├── Casual 6,000g × 2 weeks = 12,000g ✓
├── Design math is CORRECT

My simulation used 15hr/week earning 6,900g
├── Design "moderate" = 13,000g/week
├── Implies ~25-30 hr/week for "moderate"

FINDING: Player archetype definitions unclear
├── "Casual" / "Moderate" / "Hardcore" need hour definitions
├── Current: Implicit assumptions
├── Fix: Explicit hour/week definitions

RECOMMENDATION:
├── Casual: 5-10 hr/week → 4,000-6,000g/week
├── Moderate: 10-20 hr/week → 8,000-13,000g/week
├── Hardcore: 20-40 hr/week → 15,000-26,000g/week
```

```
ISSUE 3: Endgame Gold Glut

Problem:
├── Endgame players accumulate faster than sinks remove
├── After 6 months: 200,000-600,000g accumulated
├── Legendary items: 50,000-150,000g
├── Can buy 3-10 legendaries
├── Then what?

Missing Sinks for Endgame:
├── Housing/Guild halls: Not mentioned
├── Cosmetic chase: "Optional" - not compelling
├── Prestige systems: Not mentioned
├── Gold → Convenience: Limited

SEVERITY: HIGH
├── Causes inflation over server lifetime
├── Devalues progression for new players
├── Creates "nothing to do" feeling

RECOMMENDATIONS:
├── Add gold-sink cosmetic tiers (100k, 250k, 500k items)
├── Guild hall upgrades (weekly contribution goals)
├── Prestige reroll system (costly but optional)
├── Limited-time event items (gold sink events)
```

### B. Pricing Issues

```
ISSUE 4: Vendor Spread Too Wide

Current:
├── T4 Material: Player sells 150g, buys 1,000g (6.7x)
├── T4 Equipment: Player sells 800g, buys 6,000g (7.5x)

Effect:
├── Vendors are NEVER used for equipment
├── Player market mandatory for progression
├── New server: No supply, can't progress

SEVERITY: MODERATE
├── Intentional per GW1 philosophy
├── But needs supply bootstrap mechanism

RECOMMENDATION:
├── First 3 months: Vendor prices 50% reduced
├── Ensures progression without player market
├── Gradually increase to normal over months 4-6
```

```
ISSUE 5: Repair Costs Scale Poorly

Current (Implied):
├── T1 repairs: ~5g/death
├── T5 repairs: ~50g/death (estimated 10x)
├── Hardcore player: 20+ deaths/week in challenging content
├── T5 repair bill: 1,000g+/week

Analysis:
├── Design says 10% of income to repairs
├── Hardcore income: 26,000g → 2,600g repairs
├── 50g × 50 deaths = 2,500g ✓

Actually balanced correctly. Moving on.
```

```
ISSUE 6: Faction Vendor Discount Stacking

Current:
├── Exalted: 70% of base price
├── Regional specialty: -15%
├── Net: 59.5% of base price

Cross-Faction Scenario:
├── Imperial Exalted buys at 59.5%
├── Sells to Tau player at 80%
├── Profit margin: 20%

Is this a problem?
├── Requires Exalted reputation (long grind)
├── Provides role for dedicated players
├── Creates "merchant" playstyle

VERDICT: FEATURE, NOT BUG
├── This is emergent economy gameplay
├── Rewards reputation grinding
├── Creates interdependence
```

### C. Cross-System Issues

```
ISSUE 7: Combat Rewards → Economy Disconnect

Combat Document Integration:
├── Boss loot: "Personal loot (no ninja-ing)"
├── Group content: "Round-robin or FFA"
├── PvP: "50-200g based on match type"

Missing Information:
├── What's actual boss gold drop value?
├── What materials drop from bosses?
├── How often can bosses be farmed?

REQUIREMENT: Combat designer must specify:
├── Boss gold drop range (e.g., 500-1000g)
├── Boss lockout timers (e.g., daily/weekly)
├── Material drop tables and rates
├── This affects inflation calculations significantly
```

```
ISSUE 8: Crafting Economy Loop Incomplete

Current Flow:
├── Gather materials → Craft items → Sell/Use
├── Crafting fee: 10% of output value

Missing:
├── What determines "output value"?
├── Is it vendor price? Market price?
├── Market price varies - fee would vary?

RECOMMENDATION:
├── Crafting fee based on VENDOR output value
├── Fixed, predictable costs
├── Not subject to market manipulation
```

## V. DETAILED RECOMMENDATIONS

### A. Income System Adjustments

```
RECOMMENDATION 1: Implement Diminishing Returns

Current: Soft hourly cap of 500g (unclear enforcement)
Proposed: Clear diminishing returns curve

Daily Gold Income (all sources):
├── 0-4,000g: 100% rate
├── 4,001-6,000g: 50% rate
├── 6,001-8,000g: 25% rate
├── 8,001+: 10% rate

Effect:
├── Casual (4 hr): ~2,000g (no change)
├── Moderate (8 hr): ~5,000g (slight reduction)
├── Hardcore (16 hr): ~7,000g (significant reduction)
├── No-life (24 hr): ~8,500g (hard ceiling)

Preserves:
├── Reward for playing more
├── But diminishes gap between moderate/hardcore
```

### B. Sink System Adjustments

```
RECOMMENDATION 2: Add Endgame Gold Sinks

Tier 1 Sinks (100,000g range):
├── Legendary cosmetic armor sets
├── Custom title purchases
├── Personal transport customization
├── Skill visual effects

Tier 2 Sinks (250,000g range):
├── Guild hall room unlocks
├── Cross-faction reputation boost tokens
├── Rare mount unlocks
├── Weapon glow effects

Tier 3 Sinks (500,000g+ range):
├── Server-visible monuments
├── Guild hall prestige upgrades
├── Legacy achievements (account-wide)
├── Exclusive "wealthy" cosmetics

Implementation: Stagger releases
├── Month 1-3: Tier 1 sinks
├── Month 4-6: Tier 2 sinks
├── Month 7+: Tier 3 sinks
├── Matches wealth accumulation curve
```

### C. Anti-Exploit Adjustments

```
RECOMMENDATION 3: Novel Item RMT Prevention

Problem: Items without price history bypass flagging
Solution: Baseline value system

Every item has "estimated value":
├── Crafted: Sum of material vendor values × 1.5
├── Dropped: Tier baseline (T1=50g, T2=250g, etc.)
├── Unique: Manual assignment by designers

Flagging trigger:
├── Listing >300% of estimated value
├── OR listing >200% of 7-day average (if exists)
├── Whichever is MORE restrictive
```

```
RECOMMENDATION 4: Crafting Arbitrage Fix

Problem: T3 craft from vendor mats = 80g profit
Current: Materials 100g + Fee 20g = 120g → Vendor 200g

Solutions (pick one):
├── A: Reduce T3 vendor buy price to 120g
├── B: Increase crafting fee to 25% for T3 vendor sales
├── C: Add untradeable "crafting catalyst" requirement

Recommended: Option A
├── Simplest implementation
├── Maintains player market superiority
├── Closes arbitrage loop
```
