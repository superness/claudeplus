üí° Key Insights

Lessons learned from this debugging journey

üîÑ Pattern Inconsistency

The root cause was a mismatch in hook installation patterns. Crafting used synchronous immediate wrapping (worked perfectly), while mining and exploration used async setTimeout retries (caused timing issues). Consistency in patterns is critical.

üß™ Test Framework Evolution

The testing framework required 6 fixes during execution:

  • Teleport distance calibration (5000m ‚Üí 500m ‚Üí 100m)
  • API response format corrections
  • Mining range documentation updates
Each failure taught us more about the system.

üéØ Resource Type Matters

Final validation revealed mission tracker logic is resource-specific. Mining isogen doesn't trigger tritanium mission progress. Tests must verify they're collecting the exact resource type required by the mission.

‚ö° Sync vs Async Complexity

Async retry logic with setTimeout introduced race conditions and context issues. The synchronous approach is:

  • More predictable
  • Easier to debug
  • Consistent across all tracking methods

üìö Test Library Growth

Successfully added 3 regression tests to the test library, creating a new "regression" category. This builds institutional knowledge and prevents future regressions.

üîç Evidence-Driven Debugging
